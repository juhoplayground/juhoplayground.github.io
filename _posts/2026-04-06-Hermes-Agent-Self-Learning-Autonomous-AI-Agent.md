## 개요

Hermes Agent는 [Nous Research](https://nousresearch.com){:target="_blank"}가 개발한 자기 학습형 자율 AI 에이전트 플랫폼이다.
기존 IDE 코파일럿과 달리 독립적으로 동작하는 에이전트로, 사용 중 스킬을 스스로 생성하고 개선하는 자기 학습 루프를 내장하고 있다.
이 글에서는 Hermes Agent의 핵심 특징, 기술 스펙, 실행 환경 등을 정리한다.

## Hermes Agent란

Hermes Agent는 단순한 코딩 보조 도구가 아닌 자율적으로 동작하는 AI 에이전트이다.
실행 시간이 증가할수록 성능이 향상되는 구조를 가지고 있으며, 세션 간에 사용자 모델을 점진적으로 심화시킨다.
VPS, GPU 클러스터, 서버리스 인프라 등 다양한 환경에서 실행할 수 있다.

## 핵심 특징

### 자기 학습 루프

Hermes Agent의 가장 핵심적인 차별점은 자기 학습 루프를 내장하고 있다는 것이다.
에이전트가 작업을 수행하면서 새로운 스킬을 스스로 생성하고, 기존 스킬을 지속적으로 개선한다.
이를 통해 반복적인 작업에서 점점 더 효율적으로 동작하게 된다.
IDE 코파일럿처럼 사용자의 입력에 반응하는 것이 아니라, 독립적 에이전트로서 실행 시간이 길어질수록 성능이 향상되는 구조이다.

### 멀티 플랫폼 통합

Hermes Agent는 다양한 커뮤니케이션 채널을 단일 게이트웨이로 통합한다.

| 지원 플랫폼 | 유형 |
|------------|------|
| CLI | 명령줄 인터페이스 |
| Telegram | 메신저 |
| Discord | 메신저 |
| Slack | 업무 메신저 |
| WhatsApp | 메신저 |
| Signal | 메신저 |
| Email | 이메일 |

하나의 에이전트가 여러 플랫폼에서 동시에 동작하므로, 사용자는 자신이 선호하는 채널에서 에이전트와 소통할 수 있다.

### 사용자 메모리 시스템

Honcho 기반의 상태 보존형 사용자 메모리 시스템을 채택하고 있다.
FTS5 기반의 크로스 세션 리콜을 지원하여, 이전 세션에서의 대화 내용과 맥락을 다음 세션에서도 활용할 수 있다.
세션 간에 사용자 모델을 점진적으로 심화시키기 때문에, 사용할수록 사용자의 선호와 작업 패턴을 더 잘 이해하게 된다.

## 기술 스펙

### 도구 및 스킬 시스템

Hermes Agent는 40개 이상의 빌트인 도구를 제공하며, MCP 서버와의 연결도 지원한다.
[agentskills.io](https://agentskills.io){:target="_blank"} 호환 오픈 스탠더드 스킬 시스템을 사용하여, 표준화된 방식으로 스킬을 정의하고 공유할 수 있다.

| 항목 | 내용 |
|------|------|
| 빌트인 도구 | 40개 이상 |
| 번들 스킬 | 40개 이상 (MLOps, GitHub 워크플로우) |
| MCP 서버 | 연결 지원 |
| 스킬 표준 | agentskills.io 호환 |
| RL 학습 통합 | Atropos RL training |

### 실행 환경

다양한 실행 환경을 지원하여 유연한 배포가 가능하다.

| 실행 환경 | 설명 |
|----------|------|
| Docker | 컨테이너 기반 실행 |
| SSH | 원격 서버 실행 |
| Modal | 서버리스 실행 |
| Daytona | 개발 환경 실행 |
| Singularity | HPC 환경 실행 |

컨테이너 보안 측면에서는 read-only filesystems와 namespace isolation을 적용하여 안전한 실행 환경을 보장한다.

### 모델 지원

200개 이상의 모델을 지원하며, OpenRouter, OpenAI, Kimi 등 다양한 모델 제공자와 연동할 수 있다.
이를 통해 사용자는 작업 특성에 맞는 최적의 모델을 선택하여 사용할 수 있다.

## 자동화와 병렬 처리

Hermes Agent는 자동화 스케줄러를 내장하고 있으며, 자연어 cron 스케줄링을 지원한다.
사용자가 자연어로 "매일 오전 9시에 실행해줘"와 같이 지시하면, 에이전트가 이를 자동으로 cron 스케줄로 변환하여 실행한다.

병렬 처리 측면에서는 격리된 subagent를 활용하여 여러 작업을 동시에 처리할 수 있다.
각 subagent는 독립적인 환경에서 실행되므로, 작업 간 간섭 없이 안정적으로 병렬 처리가 가능하다.

## 설치 및 라이선스

설치는 단일 커맨드로 간편하게 진행할 수 있다.
uv, Python 3.11 등 필요한 의존성이 자동으로 설치된다.
Linux, macOS, WSL2 환경을 지원한다.

라이선스는 MIT 라이선스를 채택하여 완전한 오픈소스로 공개되어 있다.

## 커뮤니티 반응

커뮤니티에서는 비개발자 입장에서 다양한 에이전트 솔루션 간의 차이가 불명확하다는 의견이 있었다.
또한 "결국 모델 체급이 중요하다"는 의견도 제기되었는데, 이는 에이전트 프레임워크의 성능이 결국 기반 모델의 능력에 크게 좌우된다는 점을 지적한 것이다.

## 결론

Hermes Agent는 자기 학습 루프, 멀티 플랫폼 통합, 상태 보존형 메모리 시스템 등 차별화된 기능을 갖춘 자율 AI 에이전트 플랫폼이다.
40개 이상의 빌트인 도구와 200개 이상의 모델 지원, 다양한 실행 환경 지원 등 기술적 완성도가 높다.
MIT 라이선스 기반 오픈소스로 공개되어 있어 누구나 자유롭게 사용하고 기여할 수 있다.
자율 에이전트 분야에서 주목할 만한 프로젝트로, 향후 발전이 기대된다.
