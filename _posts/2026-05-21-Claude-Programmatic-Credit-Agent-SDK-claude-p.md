## 개요

Anthropic이 2026년 6월 15일부터 Claude 유료 플랜의 프로그래매틱 사용을 별도의 월간 크레딧으로 분리합니다.
지금까지 인터랙티브 사용과 프로그래매틱 사용이 동일한 구독 한도를 공유했지만, 앞으로는 Agent SDK, `claude -p`, GitHub Actions 사용을 위한 별도 크레딧이 부여됩니다.
변경은 Pro 20달러, Max 5x 100달러, Max 20x 200달러 플랜에 모두 적용됩니다.
이는 자동화 워크플로우의 비용 구조를 근본적으로 바꾸는 정책 변경으로, 개발자 커뮤니티에서 강한 반응을 불러일으키고 있습니다.

## 무엇이 바뀌나

핵심은 "한도의 분리"입니다.
인터랙티브와 프로그래매틱 사용이 같은 한도를 공유하던 구조가, 별도의 풀로 나뉘게 됩니다.

### 영향 범위

다음 사용 형태는 새로운 프로그래매틱 크레딧으로 분리됩니다.

- Claude Agent SDK
- `claude -p` 명령
- Claude Code GitHub Actions
- Agent SDK 기반으로 만든 서드파티 앱 (예 - Conductor, OpenClaw)

분리된 크레딧은 매월 갱신되며, 미사용 크레딧은 다음 달로 이월되지 않습니다.
크레딧 소진 후에는 유료 API 오버리지가 발생합니다.

### 영향을 받지 않는 영역

다음은 기존 구독 한도를 그대로 사용합니다.

- 인터랙티브 Claude Code 사용
- 웹과 모바일 대화
- Claude Cowork

즉, 사람이 직접 타이핑하며 대화하는 사용은 영향이 없고, 자동화·스크립트·CI로 호출되는 사용만 새로운 크레딧을 사용합니다.

## 플랜별 크레딧 구조

플랜별로 부여되는 프로그래매틱 크레딧 구조는 다음과 같습니다.

| 플랜 | 월 구독료 | 프로그래매틱 크레딧 |
|------|----------|-------------------|
| Pro | 20달러 | 별도 부여 |
| Max 5x | 100달러 | 별도 부여 |
| Max 20x | 200달러 | 별도 부여 |

구체적인 크레딧 금액은 공식 공지에서 별도로 안내될 예정입니다.
중요한 점은 "프로그래매틱 사용은 더 이상 구독 한도를 자유롭게 소진하지 못한다"는 구조 자체입니다.

## 커뮤니티 반응

발표 직후 개발자 커뮤니티에서 부정적인 의견이 쏟아졌습니다.

| 사용자 | 주요 발언 |
|--------|----------|
| @theo | "무료 크레딧으로 포장한 후퇴" |
| @mattpocockuk | "독이 든 성배 - claude -p가 10배 감소" |

일부 개발자는 `claude -p`가 사실상 종료된 것이나 마찬가지라고 평가했습니다.
다수의 댓글에서 Codex 또는 경쟁 서비스로의 마이그레이션 계획이 언급됐습니다.
자동화 워크플로우를 경제적으로 운영하기 어려워졌다는 평가가 지배적이었습니다.

## 의미와 시사점

이 변경은 Anthropic의 비용 구조와 사용자 행태 사이의 긴장을 드러냅니다.
구독 한도를 활용한 대규모 자동화는 본래 API 가격 모델로는 수익성이 맞지 않는 사용 패턴이었습니다.
한도 공유 구조 아래서 일부 사용자가 구독료의 수십 배에 해당하는 자동화 호출을 발생시키는 사례가 적지 않았던 것으로 보입니다.

반대로 사용자 입장에서는, 구독 가격에 포함된 자동화 활용도가 크게 축소된다는 점이 핵심 불만입니다.
프로그래매틱 사용이 별도 크레딧으로 격리되면, 실제로는 구독 가격 외에 추가 API 비용을 부담하는 구조에 가까워집니다.

또한 이 변경은 "어떤 사용 형태를 구독으로 묶어 팔 것인가"라는 LLM 비즈니스 모델의 본질적 질문을 다시 던집니다.
인터랙티브와 프로그래매틱 사용은 단가도, 행태도, 운영 비용도 다르며, 이를 분리해 가격을 책정하는 것이 합리적이라는 판단이 깔려 있습니다.
다만 사용자에게는 결국 "쓰던 것이 줄어든다"는 체감으로 전달돼, 정책 설계와 커뮤니케이션 사이의 간극이 그대로 드러난 사례가 됐습니다.

## 결론

2026년 6월 15일부터 Claude 유료 플랜의 프로그래매틱 사용은 별도 월간 크레딧으로 분리됩니다.
영향 받는 영역은 Agent SDK, `claude -p`, GitHub Actions, Agent SDK 기반 서드파티 앱이며, 인터랙티브 사용은 영향이 없습니다.
미사용 크레딧은 이월되지 않고, 소진 후에는 유료 API 오버리지가 발생합니다.
커뮤니티는 자동화 비용 부담 증가를 이유로 부정적으로 반응하고 있으며, Codex 등 경쟁 서비스로의 이동을 검토하는 의견이 나오고 있습니다.
자동화 워크플로우를 운영 중이라면, 새로운 크레딧 한도 안에서 사용 패턴을 재설계하는 것이 우선 과제가 될 전망입니다.
