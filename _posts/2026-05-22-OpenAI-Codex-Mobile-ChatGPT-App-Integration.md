## 개요

OpenAI가 Codex를 iOS 및 Android의 ChatGPT 모바일 앱에 통합했다.
중요한 점은 이것이 "폰에서 코드를 작성하는" 기능이 아니라는 것이다.
데스크톱에서 실행 중인 Codex 세션을 모바일로 원격 관리하는 콘셉트다.

## "폰에서 코딩"이 아닌 원격 세션 관리

모바일 환경에서 직접 코드를 작성하는 것은 입력 인터페이스의 한계로 비현실적이다.
OpenAI는 이를 인정하고 다른 방향을 택했다.
데스크톱에서 돌고 있는 Codex 에이전트 세션을 모바일에서 들여다보고 통제하는 형태다.

기존에 사용자는 노트북을 열어둔 채 외출하거나, 장시간 실행되는 에이전트 작업이 끝날 때까지 기다려야 했다.
모바일 통합은 이 문제를 해결한다.
이동 중에도 작업이 어디까지 진행됐는지 확인하고, 막힌 부분에 응답할 수 있다.

## 주요 기능

### 세션 모니터링과 승인

실행 중인 작업 스레드를 모바일에서 직접 확인할 수 있다.
에이전트가 도구 호출이나 파일 변경에 대한 승인을 요청하면 모바일에서 응답 가능하다.
이는 사람이 개입하는 HITL(Human-in-the-loop) 흐름을 모바일로 확장한 형태다.

### 모델 변경과 새 작업 시작

진행 중인 세션에서 사용 모델을 변경하거나 완전히 새로운 작업을 시작할 수 있다.
복잡한 추론이 필요한 시점에는 더 강력한 모델로 전환하고, 단순 반복 작업에는 가벼운 모델을 쓰는 식의 운영이 모바일에서 가능해진다.

### 실시간 출력 확인

터미널 출력, 테스트 결과, 스크린샷 등 에이전트의 작업 산출물을 실시간으로 본다.
무엇이 잘 되고 있는지, 어디서 막혔는지를 화면에서 즉시 파악할 수 있다.

## 장시간 에이전트 작업이라는 새 패턴

이번 통합의 진짜 의미는 기능 자체보다 OpenAI가 "장시간 실행되는 에이전트 작업"을 핵심 사용 패턴으로 인정했다는 점이다.

| 패턴 | 작업 시간 | 사용자 개입 |
|------|----------|------------|
| 전통 코딩 도구 | 수초 ~ 수분 | 지속적 |
| 일반 AI 어시스턴트 | 수분 | 빈번 |
| 장시간 에이전트 | 수십 분 ~ 수시간 | 산발적 승인 |

장시간 에이전트는 사용자가 계속 화면을 바라보는 가정에서 벗어난다.
백그라운드에서 진행되고, 필요할 때만 사람을 부른다.
모바일 통합은 이 시나리오에 맞춘 인프라다.

## 현재 상태와 사용자 반응

현재 macOS 중심으로 배포되고 있으며 Windows 지원도 예정되어 있다.
일부 사용자는 "Request is not allowed" 오류를 보고하고 있으나 점진적으로 해소되는 분위기다.

커뮤니티 반응은 대체로 긍정적이다.
편의성 증대와 데스크톱 세션과의 동기화 용이성이 호평을 받는다.
Claude Code와의 경쟁 속에서 OpenAI가 빠르게 기능을 확장하고 있다는 평가도 나온다.
일부에서는 Anthropic의 원격 기능보다 더 안정적이라는 의견도 있다.

## 결론

OpenAI Codex Mobile은 모바일에서 코드를 직접 짜는 도구가 아니다.
데스크톱 에이전트 세션을 원격으로 모니터링·승인·재구성하는 컨트롤 패널이다.
이는 코딩 에이전트의 사용 패턴이 "옆에 앉아 지켜보는 도구"에서 "백그라운드에서 알아서 일하는 동료"로 옮겨가고 있다는 신호다.
"폰에서 코딩"이라는 표현보다 "에이전트 운영"이라는 표현이 적절한 시대가 시작됐다.
