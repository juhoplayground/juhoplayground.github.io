## 개요

Cursor 3.0이 2026년 4월 2일에 공개되었다.
기존 IDE 중심 방식에서 탈피하여 에이전트 기반 워크스페이스로 완전히 재설계된 것이 핵심이다.
단순한 자동완성 도구를 넘어 다중 에이전트를 관리하고 조율하는 개발 환경으로 진화했다.

## 핵심 기능

### Agents Window

로컬, 워크트리, 클라우드, 원격 SSH 등 다양한 환경에서 여러 에이전트를 병렬로 실행할 수 있는 새로운 인터페이스이다.
더 단순하면서도 강력한 구조로 설계되었다.
`Cmd+Shift+P`에서 Agents Window를 통해 접근할 수 있다.

### Design Mode

에이전트 창에서 브라우저 UI 요소에 직접 주석을 달고 대상을 지정하여 에이전트에 작업을 전달할 수 있다.
정밀한 시각적 피드백이 가능하다.

주요 단축키이다.

| 단축키 | 기능 |
|--------|------|
| Cmd + Shift + D | 디자인 모드 전환 |
| Shift + 드래그 | 영역 선택 |
| Cmd + L | 요소를 채팅에 추가 |
| Alt + 클릭 | 요소를 입력창에 추가 |

UI 작업에서 "이 버튼의 색상을 바꿔줘"와 같은 지시를 시각적으로 정확하게 전달할 수 있다.

### Agent 탭

여러 에이전트와의 대화를 나란히 또는 격자 형태로 관리할 수 있다.
동시에 여러 작업을 진행하면서 각 에이전트의 상태를 한눈에 파악할 수 있다.

## 에디터 개선

4가지 주요 에디터 개선이 포함되었다.

| 기능 | 설명 |
|------|------|
| /worktree | 격리된 git 워크트리에서 변경 사항 적용 |
| /best-of-n | 여러 모델의 결과를 병렬로 비교 |
| 워크트리/best-of-n 선택 | 기존 선택 기능 단계적 폐지 |
| 클라우드 에이전트 | 에디터에서 클라우드 에이전트 제거 |

`/worktree` 명령어는 메인 브랜치에 영향을 주지 않고 격리된 환경에서 변경 사항을 적용할 수 있어, 실험적인 작업에 유용하다.
`/best-of-n`은 동일한 프롬프트를 여러 모델에 동시에 보내 결과를 비교하는 기능이다.

## 플러그인 및 MCP 강화

MCP(Model Context Protocol) 앱이 구조화된 콘텐츠를 지원하여 더 풍부한 도구 출력이 가능해졌다.
기업 플랜에서는 타사 플러그인이 기본적으로 비활성화되어 보안이 강화되었다.

## 기업 기능

기업 환경을 위한 3가지 개선이 추가되었다.

| 기능 | 설명 |
|------|------|
| 감사 로그 | 디렉터리 그룹 이름 추가 |
| 팀 시크릿 | 관리 권한 세분화 |
| 브랜딩 | "Made with Cursor" 표시 조직 단위 비활성화 옵션 |

## 기타 개선 및 버그 수정

10가지 일반 개선 사항이 포함되었다.
대용량 파일 diff 렌더링 성능 향상, 장시간 작업 모니터링 개선, Await 도구 추가, 브라우저 자동화 개선 등이 주요 항목이다.
캐싱 추가로 응답 속도가 향상되었고, at-mention 검색이 개선되었다.

8가지 버그 수정도 이루어졌다.
텍스트 영역 동작, 다중 루트 워크스페이스 hook 로딩, markdown 파싱, thinking 블록 펼치기/접기 등이 수정되었다.

## 의미와 시사점

Cursor 3.0은 AI 코딩 도구의 진화 방향을 보여준다.
단일 에이전트 대화에서 다중 에이전트 관리로의 전환은, IDE가 개발자의 코드 편집기에서 AI 에이전트 팀의 관리 콘솔로 변화하고 있음을 의미한다.
Design Mode는 비개발자도 시각적으로 UI 수정을 지시할 수 있게 하여, 개발 접근성을 높인다.
`/worktree`와 `/best-of-n` 같은 명령어는 에이전트 작업의 안전성과 품질을 동시에 높이는 접근이다.

## 결론

Cursor 3.0은 에이전트 중심 개발 환경으로의 전환을 선언한 의미 있는 업데이트이다.
Agents Window, Design Mode, Agent 탭 등 새로운 기능들은 다중 에이전트를 효과적으로 관리하는 데 초점을 맞추고 있다.
AI 코딩 도구가 단순한 코드 자동완성을 넘어 개발 프로세스 전체를 재정의하는 단계에 진입했음을 보여준다.
