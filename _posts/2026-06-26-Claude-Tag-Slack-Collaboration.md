## 개요

Anthropic이 2026년 6월 23일 Claude Tag를 공개했다.
Claude Tag는 Slack에 통합되어 팀이 Claude를 멘션해 작업을 위임할 수 있는 협업 AI 기능이다.
사용자는 채널에서 @Claude를 태그해 작업을 맡기고, Claude는 채널 대화와 연결된 도구에서 컨텍스트를 학습한다.
Claude Code의 발전형으로, Claude를 더 능동적으로 만들고 팀 단위 협업에 더 적합하도록 설계했다.
현재 Claude Enterprise와 Claude Team 고객을 대상으로 베타로 제공된다.

## 배경

기존에는 Claude in Slack 앱을 통해 개별 사용자가 Claude와 상호작용했다.
Claude Tag는 이 기존 Claude in Slack 앱을 대체한다.
Anthropic은 Claude를 한 명의 팀 구성원처럼 Slack 워크스페이스에 합류시키는 방향으로 협업 모델을 확장했다.
이를 통해 개인 단위의 도구 사용을 넘어 팀 전체가 하나의 Claude 인스턴스와 함께 일하는 멀티플레이어 환경을 제공한다.

## 핵심 내용

Claude Tag의 작동 방식과 기능을 항목별로 정리한다.

### Slack 통합 방식

Claude는 Slack 워크스페이스에 팀 구성원으로 합류한다.
사용자는 채널에서 @Claude를 태그해 작업을 위임한다.
채널 안의 누구나 @Claude를 태그해 작업을 맡길 수 있다.
하나의 채널 안에서는 하나의 Claude 인스턴스가 모든 구성원과 상호작용한다.
팀 구성원은 진행 상황을 함께 관찰하고 대화를 자연스럽게 이어갈 수 있다.
또한 사용자는 개인 도구와 커넥터를 사용해 Claude에게 비공개 다이렉트 메시지를 보낼 수 있다.

### 주요 기능

Claude Tag는 멀티플레이어, 컨텍스트 학습, 능동적 지원, 비동기 작업을 핵심 기능으로 제공한다.
다음은 각 기능을 정리한 표다.

| 기능 | 설명 |
|------|------|
| 멀티플레이어 | 채널 내 단일 Claude 인스턴스가 모든 구성원과 상호작용하며 팀이 진행 상황을 함께 관찰한다 |
| 컨텍스트 학습 | 채널 토론을 따라가며 작업에 대한 컨텍스트를 쌓고, 적절한 권한으로 연결된 채널과 데이터 소스에서 학습한다 |
| 능동적 지원 | 앰비언트 동작을 활성화하면 관련 정보를 짚어주고 미해결 스레드를 자율적으로 후속 처리한다 |
| 비동기 작업 | 사용자가 다른 일에 집중하는 동안 작업을 수행하며, 스스로 작업을 예약해 프로젝트를 자율적으로 진행한다 |

컨텍스트 학습은 Claude가 채널 토론을 따라가면서 작업에 대한 맥락을 더 깊이 쌓는 방식으로 동작한다.
능동적 지원은 앰비언트 동작이 켜졌을 때 Claude가 먼저 관련 정보를 알려주고 해결되지 않은 스레드를 자율적으로 이어가는 것을 의미한다.
비동기 작업에서는 Claude가 스스로 작업을 예약하고 프로젝트를 자율적으로 추진할 수 있다.

### 접근 권한과 보안

관리자는 채널별로 Claude가 접근하는 도구와 데이터를 제어한다.
용도에 따라 별도의 Claude 아이덴티티를 둘 수 있다.
데이터와 메모리는 정의된 채널 범위 안으로 한정된다.
관리자는 토큰 지출 한도를 설정하고 활동 로그를 확인할 수 있다.
다음은 접근 권한과 보안 항목을 정리한 표다.

| 항목 | 내용 |
|------|------|
| 도구 및 데이터 제어 | 관리자가 채널별로 Claude의 접근 범위를 설정한다 |
| Claude 아이덴티티 | 용도에 따라 분리된 아이덴티티를 운영할 수 있다 |
| 데이터 범위 | 데이터와 메모리가 정의된 채널 안으로 한정된다 |
| 지출과 감사 | 토큰 지출 한도 설정과 활동 로그 조회가 가능하다 |

### 활용 사례

Anthropic 내부에서는 제품팀 코드의 65%가 Claude Tag의 내부 버전으로 작성된다.
활용 범위는 엔지니어링에 그치지 않는다.
지표 분석, 지원 티켓 처리, 디버깅 등으로 사용이 확장되고 있다.

## 의미와 시사점

Claude Tag는 AI를 개인 도구에서 팀 단위 협업 구성원으로 확장한다.
채널에 합류한 단일 인스턴스가 팀 전체의 맥락을 공유하므로, 작업 위임과 진행 상황 관찰이 한 공간에서 이뤄진다.
관리자 단위의 도구 제어, 채널 범위 데이터 한정, 지출 한도, 활동 로그는 엔터프라이즈 도입에 필요한 통제 장치를 제공한다.
내부 제품팀 코드의 65%가 자동 생성된다는 수치는 실제 업무 흐름에 깊이 결합될 수 있음을 보여준다.

## 결론

Claude Tag는 Opus 4.8 모델로 동작하며 2026년 6월 23일 출시됐다.
설정은 Slack과 페어링, 도구 접근 권한 부여, 지출 한도 설정, 테스트의 네 단계로 이뤄진다.
자격을 갖춘 조직에는 시험 사용을 위한 런치 크레딧이 제공된다.
현재 Claude Enterprise와 Claude Team 고객에게 베타로 열려 있으며, 기존 Claude in Slack 앱을 대체한다.
